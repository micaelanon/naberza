# naBerza · notas rápidas de UX/frontend

## Diagnóstico honesto

La UI actual se siente genérica por varias razones:

1. **hero demasiado genérico**
   - titular amplio y bonito, pero poco operativo
   - se parece más a landing que a herramienta diaria

2. **stats decorativas**
   - pendientes / persistentes / hechas no ayudan mucho a decidir qué hacer
   - ocupan espacio premium sin empujar acción

3. **cards homogéneas**
   - todas pesan igual
   - no hay foco claro en la tarea importante

4. **copy poco específico**
   - "Tus pendientes, sin ruido" está bien como intención, pero no da identidad suficiente
   - falta lenguaje más concreto alrededor de recordatorios persistentes

5. **demasiado scaffold de Next**
   - estructura correcta, pero sin gesto de producto
   - limpio, sí; memorable o útil de verdad, todavía no

## Dirección recomendada

Ir hacia una mezcla de:
- **Things** para calma y jerarquía
- **Todoist** para claridad operativa
- **Twos** para captura ligera y cercanía

Pero sin copiar ninguna.

## Qué haría en la siguiente iteración

### 1. cambiar el home
Quitar el hero grande y pasar a una home más de trabajo:
- bloque principal: `Hoy`
- bloque secundario: `Persistentes`
- bloque terciario: `Más tarde`

### 2. dar protagonismo a persistentes
Las tareas persistentes son la parte diferencial.
Deben verse como un carril propio, no como una card más.

### 3. meter una acción principal arriba
Ejemplo:
- `Añadir tarea`
- `Añadir recordatorio persistente`

### 4. reducir decoración y subir criterio
Menos cristal y más estructura.
Menos "dashboard" y más lista viva.

### 5. mejorar tono visual
Paleta más intencional:
- piedra / crema / musgo / tinta
- menos look SaaS genérico
- más producto personal cuidado
