# Copilot Instructions for naBerza

## Naming Conventions

### Directories
- **All lowercase**, no underscores, no hyphens
- Use clear, singular nouns
- Examples: `components`, `lib`, `hooks`, `utils`, `types`, `pages`

### NO ALLOWED:
- ❌ `_components` (leading underscore)
- ❌ `(dashboard)` (route groups)
- ❌ `task-card` (hyphens in folder names)
- ❌ `TaskCard` (PascalCase directories)

### Files
- **Components**: PascalCase (e.g., `TaskCard.tsx`, `Button.tsx`)
- **Utilities/Hooks**: camelCase (e.g., `useTaskForm.ts`, `formatDate.ts`)
- **Types**: PascalCase (e.g., `Task.ts`, `User.ts`)
- **Styles**: match file name + `.css` (e.g., `TaskCard.css`)

## Rules

1. **No route groups**: Don't use `(name)` folders for organization
2. **Flat structure preferred**: Avoid deep nesting
3. **Colocation**: Keep related files together (component + styles + types)
4. **Imports**: Use `@/` alias for absolute imports
5. **Components are PascalCase files**, directories are lowercase

## Breaking from this

If you need to break a rule, explicitly ask user approval first.
